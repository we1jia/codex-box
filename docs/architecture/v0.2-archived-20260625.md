# Codex Box · 架构方案 v0.2

> 状态：v0.2 基线 · 2026-06-25
> 目标：把 OpenCodex 的核心能力复现到 Codex Box，同时保留独立 UI、独立代码路径和安全边界。

---

## 1. 一句话架构

Codex Box 是一个 Tauri 桌面控制台，前端负责 Mac Dashboard 体验，Rust 后端负责配置读写、诊断、进程管理和 gateway runtime，gateway runtime 负责 HTTP / WebSocket、访问密码、日志、health 和移动端访问。

```
Codex Box (Tauri)
  ├─ React UI
  │  ├─ Overview
  │  ├─ Gateway
  │  ├─ Mobile Access
  │  ├─ Codex Runtime
  │  ├─ Profiles
  │  ├─ Providers
  │  ├─ Diagnostics
  │  └─ Settings
  ├─ Rust Backend
  │  ├─ commands/
  │  ├─ config/
  │  ├─ gateway_runtime/
  │  ├─ health/
  │  └─ secret/
  ├─ ~/.codex/config.toml
  ├─ ~/.codex/codex-box/
  └─ Codex Desktop / Codex CLI
```

---

## 2. 分层职责

| 层 | 职责 | 不做什么 |
|---|---|---|
| React UI | 页面、导航、表单、diff 确认、状态展示 | 不直接读写用户文件 |
| Tauri commands | 前后端命令边界、参数校验、错误脱敏 | 不承载复杂业务状态 |
| config core | TOML 读取、解析、diff、backup、atomic write、rollback | 不做 UI 决策 |
| gateway runtime | host / port、HTTP / WS、auth、health、log、LAN 访问 | 不抓 token、不改 Codex Desktop |
| health | config / runtime / provider / MCP / network 诊断 | 不输出 secret 明文 |
| secret | env / Keychain 引用和脱敏 | 不持久化 token 明文 |

---

## 3. 页面结构

主导航收敛为 8 个页面：

| 页面 | 主要能力 |
|---|---|
| Overview | 全局状态、活跃 profile、gateway 状态、快速入口 |
| Gateway | runtime 启停、host、port、health、日志 |
| Mobile Access | LAN URL、二维码、密码状态、安全提示 |
| Codex Runtime | Codex Desktop / CLI / CODEX_HOME 检测 |
| Profiles | 工作配置管理 |
| Providers | 模型来源管理 |
| Diagnostics | 脱敏诊断报告 |
| Settings | 应用设置、日志、备份、实验开关 |

`MCP / Network / Config Diff` 作为能力区块放在相关页面中，稳定后再拆成独立页面。

## 4. 核心数据流

### 4.1 Dashboard 读取

```
Overview 挂载
  -> invoke("dashboard_summary")
  -> 读取 config_snapshot
  -> 读取 gateway_status
  -> 读取 codex_runtime_status
  -> 聚合 health 摘要
  -> 渲染状态卡
```

### 4.2 配置写入

```
用户发起变更
  -> preview_config_change
  -> 生成 diff + warnings + current_hash
  -> 用户确认
  -> apply_config_change(expected_hash)
  -> backup
  -> atomic write
  -> 失败 rollback
  -> 返回新 snapshot
```

### 4.3 Gateway 启动

```
用户点击启动
  -> 校验 host / port / password policy
  -> 准备 runtime_dir / log_path / config_path
  -> 启动 embedded gateway runtime
  -> probe /api/health
  -> 返回 GatewayRuntime
```

### 4.4 Mobile Access

```
用户启用 LAN
  -> 必须先配置访问密码
  -> host 切换到 0.0.0.0 或显式 LAN host
  -> 生成 LAN URL / QR Code
  -> 登录失败 rate limit
```

## 5. OpenCodex 复现策略

Codex Box 要复现的是 OpenCodex 的功能结果，而不是复制其代码和 UI：

- gateway 生命周期管理
- 本机 / LAN 访问地址
- 访问密码和 session
- health endpoint
- 日志轮转和脱敏
- Codex Desktop / Codex CLI / CODEX_HOME 状态检测
- 移动端访问体验

过渡期允许保留 `ExternalOpenCodexCheckout` 启动能力，用于对照验证；长期目标是移除硬编码外部路径并收敛为 `Embedded` runtime。

---

## 6. 安全边界

- 默认监听 `127.0.0.1`
- LAN 模式必须有访问密码
- 不抓取、不复用、不转发官方账号 token
- 不绕过官方登录
- 不规避 rate limit / 账号配额
- 不默认修改 Codex Desktop 内部文件
- 不接管系统全局代理
- 日志不记录 prompt、文件正文、API key、token、password

---

## 7. 模块落地顺序

1. `gateway_runtime/settings.rs`：读取/写入 gateway 设置
2. `gateway_runtime/access.rs`：密码状态、LAN 策略、URL 生成
3. `gateway_runtime/process.rs`：启动、停止、重启、端口检测
4. `gateway_runtime/health.rs`：`/api/health` probe
5. `gateway_runtime/logs.rs`：日志路径、tail、轮转策略
6. `codex_runtime/locator.rs`：Codex Desktop / CLI / CODEX_HOME 检测
7. `commands/gateway.rs`：Tauri command 边界
8. `commands/diagnostics.rs`：诊断聚合
