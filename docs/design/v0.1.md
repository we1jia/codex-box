### 2.2 字体

> **v0.6 修订**:中文回退到系统字体,英文 / 等宽继续走 webfont。详见 [ADR-0003](../decisions/0003-中文字体回退系统字体.md)。
> 修订前(v0.5)曾引入 Noto Sans SC / Noto Serif SC 解决跨平台一致 + 衬线大标题字重问题,见 [ADR-0001](../decisions/0001-字体方案.md)。bundle 减重约 3.7MB 原始 / ~2.5MB gzipped,代价是跨设备中文表现回归"系统差异"+ 衬线大标题 `font-semibold` 在中文部分不再视觉生效。

字体家族:

```css
/* 中文衬线大标题(品牌感) — 系统字体栈:
 *   macOS  :Songti SC / STSong
 *   Windows:SimSun / FangSong
 *   Linux  :Noto Serif CJK SC
 * 注意:macOS / Windows 系统宋体只有 Regular 一档,font-semibold 在中文部分
 * 不再视觉生效,英文 / 数字仍走 Inter Variable 600(见 ADR-0003 权衡) */
--font-serif: 'Songti SC', 'STSong', 'STSongti-SC-Regular', 'Source Han Serif SC',
             'SimSun', 'FangSong', 'Noto Serif CJK SC', 'AR PL UMing CN',
             'Times New Roman', serif;

/* 正文 — 英文 Inter Variable(可变字体,100-900 全字重,本地 webfont),
   中文走系统无衬线(覆盖三大平台):
     macOS  :PingFang SC / Hiragino Sans GB
     Windows:Microsoft YaHei
     Linux  :Noto Sans CJK SC / WenQuanYi Micro Hei
   浏览器按字符 unicode-range 自动选择字体,无需手动切换 */
--font-sans: 'Inter Variable', 'Inter', 'PingFang SC', 'Hiragino Sans GB',
             'Microsoft YaHei', 'Noto Sans CJK SC', 'WenQuanYi Micro Hei',
             system-ui, sans-serif;

/* 等宽(代码 / diff / log) — 继续 webfont */
--font-mono: 'JetBrains Mono Variable', 'JetBrains Mono', 'SF Mono', 'Menlo', monospace;
```

字号 / 字重梯度:

| 用途 | 字号 | 字重 |
|---|---|---|
| 衬线大标题("晚安,开发者") | 32px | **600**(Noto Serif SC 含 600) |
| 卡片标题 | 18px | 600 |
| 指标主数值 | 24px | 600 |
| 正文 | 14px | 400 |
| 副文 / 状态 | 12px | 400 |
| Badge / 键盘快捷键 / label | 11px | 500 |

字距 token(letter-spacing,中英文分别处理):

| token | 值 | 用途 |
|---|---|---|
| `tight-cn` | -0.005em | 中文标题收紧 |
| `normal-cn` | 0 | 中文正文默认 |
| `wide-cn` | 0.02em | 中文副文 |
| `wide-cn-lg` | 0.05em | 中文 label(替代 uppercase) |
| `tight-en` | -0.011em | 英文标题(Inter 推荐) |
| `normal-en` | -0.005em | 英文正文 |
| `wide-en` | 0.025em | 英文副文 |
| `wider-en` | 0.05em | 英文 label / uppercase |

行高 token(line-height,中文偏松,英文偏紧):

| token | 值 | 用途 |
|---|---|---|
| `tight-cn` | 1.25 | 中文标题 |
| `snug-cn` | 1.5 | 中文卡片标题 |
| `normal-cn` | 1.7 | **中文正文(全局默认)** |
| `relaxed-cn` | 1.8 | 中文长段落 |
| `tight-en` | 1.15 | 英文标题 |
| `snug-en` | 1.35 | **英文正文** |
| `normal-en` | 1.5 | 英文长段落 |

OpenType 特性(`body` 全局默认):

- `tnum 1`:数字等宽(对齐表格数字)
- `ss01 1`:Inter 双层 'a'
- `cv11 0`:关闭 Inter single-storey 'a'(避免与数字混淆)
- `calt 1` + `liga 1`:默认连字

语义化 utility class(定义在 `src/index.css`):

| class | 作用 |
|---|---|
| `.en-tight` | 英文段落收紧 + 偏紧行高 |
| `.cn-relaxed` | 中文段落略宽字距 + 偏松行高 |
| `.num-tab` | 数字串专用(强制 tabular-nums + cv11 off) |
| `.title-display` | 衬线大标题(收紧英文 + 紧行高) |

label 类元素(11px / 中英兼容):用 `tracking-wide-cn-lg` 而非 `uppercase tracking-wider`,
后者对中文无效且视觉过宽。

## 3. 布局规则

### 3.1 Window

- 窗口：标准 macOS 红黄绿 traffic lights
- 标题栏居中："Codex Box"
- 右上角：搜索 ⌘K、设置齿轮、用户头像

### 3.2 Sidebar（独立悬浮玻璃面板）

- 位置：左侧，距窗口左/上/下边缘各 **24~28px gap**
- 宽度：**260~280px**
- 背景：`rgba(255,255,255,0.72)` + `backdrop-blur(20px)`
- 圆角：16px
- 阴影：双层柔和阴影
- 描边：1px `rgba(255,255,255,0.6)`
- **与主内容区之间留 20~24px gap**

Sidebar 内部结构（从上到下）：

| 区块 | 内容 |
|---|---|
| 品牌区 | ◆ Codex Box（衬线小 logo） + ◉ connected · v0.3（状态徽章） |
| 主导航 | 8 项（Overview / Models / Provider Routes / Codex Runtime / Profiles / Diagnostics / Settings）+ ⌘N 快捷键（v0.3 起把 Gateway / Mobile Access 合并为 Models + Provider Routes） |
| 分组区 | 极小灰色"分组"标题 + 4 项（profiles / providers / network / mcp），每项带副文字 |
| 底部 | 圆形头像 CB + 安全/主动模式切换 |

### 3.3 主内容区（分层卡片）

- 主内容区与窗口边缘四周各留 24~28px padding
- 卡片之间 16~20px gap
- 每张卡片独立圆角 + 阴影

Dashboard 主内容区从上到下：

| 顺序 | 卡片 | 内容 |
|---|---|---|
| 1 | 欢迎区 | 大标题"晚安，开发者" + 副标题 + Daily/Settings chip |
| 2 | 启动设置 | 横向 3 步骤卡 + 0/5 badge |
| 3 | 配置健康 | 检查项表格 + 3 张圆形指标卡（pass / direct / 8/8） |
| 4 | 最近活动 + 快速操作 | 时间线 + 三按钮（双卡左右布局） |

### 3.4 悬浮贴纸（错位叠加）

- 位置：主内容区右上角
- 数量：2 张
- **错位**：上下错开约 40px
- **部分超出主卡右边界**
- 阴影比主卡强（明显"贴上去"的浮起感）
- 贴纸内容：
  - TOKEN 成本：`$0.402` / `7d cost · 19K tokens` / 折线
  - 收入：`$0` / `No payments yet` / 绿色折线

## 6. 交互规范

- **快捷键**：⌘1~⌘7 切换 7 个主页面，⌘, 打开 Settings，⌘K 打开搜索
- **修改前必 diff**：所有写操作必须先展示 diff
- **危险操作二次确认**：删除 profile、删除 provider、删除 MCP server、回滚 config
- **toast 反馈**：成功 / 失败 / 警告统一用右下角 toast
- **键盘可达性**：所有交互元素支持 Tab 导航
- **空状态**：每个列表页都有"空状态"插画 + 引导文案
- **加载状态**：骨架屏，不用 spinner
- **错误状态**：友好提示 + "复制诊断"按钮
