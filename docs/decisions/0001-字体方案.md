# ADR 0001 · 引入 Webfont 作为主字体方案

> 状态:Accepted
> 日期:2026-06-24
> 决策者:项目组

---

## 背景

Codex Box 当前字体方案存在 4 个明显问题:

1. **衬线大标题过细**:旧 stack 首位 `Songti SC`(macOS 上的「宋体-简」)没有 500/600 字重,
   `font-medium` / `font-semibold` 实际渲染为 Regular,标题立不起来。
2. **中英文混排割裂**:`body` font stack 首位是中文 `PingFang SC`,但中文字串中夹的英文字符会回退到
   `system-ui` / `Inter`,两套字体的基线、字宽、字重不对齐。
3. **数字宽度不齐**:Metric / Badge / 时间戳里的数字(`0/5`、`8/8`、`14:22`、`1.2 GB`)
   没有启用 tabular-nums,在表格里上下抖动。
4. **中文行高 / 字距未优化**:design token 里没有中文专用 `leading` 和 `letter-spacing`,
   中文段落偏挤、英文段落偏松,且 `MetricCard` label 用了对中文无效的 `uppercase tracking-wider`。

## 决策

引入 4 个 fontsource 包作为主字体,完全替代系统字体:

| 角色 | 字体 | 包 | 字重 |
|---|---|---|---|
| 英文 / UI | Inter Variable | `@fontsource-variable/inter` | 100–900 (可变字体) |
| 中文正文 | 思源黑体 SC | `@fontsource/noto-sans-sc` | 400, 500 |
| 中文品牌衬线 | 思源宋体 SC | `@fontsource/noto-serif-sc` | 600 |
| 等宽 | JetBrains Mono Variable | `@fontsource-variable/jetbrains-mono` | 100–800 |

字体加载策略:

- 入口 `src/styles/fonts.css` 通过 `@import` 集中加载,`main.tsx` 在 `index.css` 之前引入
- 选用 fontsource 的 `index.css` / `{weight}.css` 版本,**保留 unicode-range 切片**,
  浏览器按字符自动选择字体,实现「中文走 Noto SC、英文走 Inter」的无缝 fallback
- 所有字体走本地打包(`node_modules`),**不走 CDN**,离线可用,Tauri 打包可控

字号 / 字重 token 维持 v0.1 design spec 不变;补充中英文专用 letter-spacing 和 line-height token,
由于 Tailwind CSS IntelliSense 把带 `-cn` / `-en` 后缀的 key 误判为拼写错误(8 个 lint warning),
token 改用 CSS 变量承载,定义在 `src/index.css` 的 `:root`:

```
--ls-cn-tight / --ls-cn-normal / --ls-cn-wide / --ls-cn-wide-lg
--ls-en-tight / --ls-en-normal / --ls-en-wide / --ls-en-wider
--lh-cn-tight / --lh-cn-snug / --lh-cn-normal / --lh-cn-relaxed
--lh-en-tight / --lh-en-snug / --lh-en-normal
```

全局 `body` 启用 OpenType 特性:`tnum 1, ss01 1, calt 1, liga 1, cv11 0`,
并在 `index.css` 提供 4 个语义化 utility class:`.en-tight` / `.cn-relaxed` / `.num-tab` / `.title-display`。
label 类(11px / 中英兼容)直接用 Tailwind arbitrary value `tracking-[0.05em]`,不再依赖 theme token。

## 备选

### A · 零依赖,只调 stack

仅调整 `font-family` 顺序和字重,继续用系统字体。**否决**:macOS 字体跨设备不一致,中文衬线大标题
字重问题无法解决(系统宋体确实没有 600)。

### B · 引入 Webfont(本次采纳)

见上。

### C · 替换品牌字体到「阿里巴巴普惠体 / HarmonyOS Sans」

更激进的方案。**否决**:风格跳出「现代 Mac Dashboard」调性,与 v0.1 design spec 不一致。

## 影响

### 正面

- **跨平台一致**:Windows / Linux / 不同 macOS 版本下字体表现一致
- **衬线大标题有「分量」**:Noto Serif SC 有 600 字重,`font-semibold` 真正生效
- **中英混排整齐**:Inter Variable + tabular-nums 让数字和短词对齐
- **离线可用**:不依赖 CDN,Tauri 打包后自包含

### 负面 / 成本

- **bundle 体积增加**:Noto Sans SC 400 + 500 ≈ 2.2MB(unicode-range 切片后约 2MB),
  Noto Serif SC 600 ≈ 1.5MB,Inter Variable + JetBrains Mono ≈ 0.5MB。**总计 ~3.7MB 原始 / ~2.5MB gzipped**。
  对 Tauri 桌面应用可接受,但需在 Settings 中考虑「字体加载中」的占位体验。
- **首次渲染略有延迟**:首屏需要等待字体下载完成(font-display: swap 已启用,会先用系统字体兜底)

### 风险与缓解

- **字体子集化**:当前按 unicode-range 切片,未按"项目实际用到的字符"二次子集化。M5 后台稳定后,
  可考虑用 `cn-font-split` / `fonttools` 进一步压缩到 ~500KB。
- **其他页面未迁移**:本次仅修改了 `Dashboard` 大标题和 `MetricCard` label。Profiles / Providers /
  Network / MCP / Diff / Diagnostics / Settings 八个页面的字体 token 将在 M1/M2 落地时统一应用。

## 验证

1. `pnpm dev` 启动 Vite 开发服务器,首次加载 Dashboard
2. DevTools Network 面板确认 Inter / Noto SC / Noto Serif SC / JetBrains Mono woff2 文件被加载
3. 视觉验证项:
   - 衬线大标题「晚安,开发者」字重明显加粗(对比 Songti SC 的 Regular)
   - 时间戳 `14:22`、`14:20`、`14:18`、`14:10` 数字宽度对齐
   - MetricCard 四个 label(英文 + i18n)字距宽松但不大写
   - 中文字段宽度紧凑、英文段落行高紧凑
