## 1. 总览

Codex Box v0.2 围绕三组模型构建：

1. **Codex 配置模型**：`CodexConfigSnapshot`、`CodexProfile`、`ModelProvider`、`McpServer`、`NetworkRoute`
2. **Gateway / Runtime 模型**：`GatewayRuntime`、`GatewayAccess`、`CodexRuntimeStatus`
3. **安全写入与诊断模型**：`ConfigChangePreview`、`BackupRecord`、`HealthStatus`、`SecretRef`

```
CodexConfigSnapshot
  ├─ CodexProfile
  ├─ ModelProvider
  ├─ McpServer
  └─ NetworkRoute

GatewayRuntime
  ├─ GatewayAccess
  └─ CodexRuntimeStatus

ConfigChangePreview
  ├─ BackupRecord
  ├─ HealthStatus
  └─ SecretRef
```

## 2. CodexConfigSnapshot

```rust
struct CodexConfigSnapshot {
    path: PathBuf,
    read_at: DateTime<Utc>,
    raw_text: String,
    parsed: ParsedConfig,
    content_hash: String,
    valid: bool,
    parse_errors: Vec<ParseError>,
}

struct ParsedConfig {
    active_profile: Option<String>,
    profiles: Vec<CodexProfile>,
    model_providers: Vec<ModelProvider>,
    mcp_servers: Vec<McpServer>,
    default_network: Option<String>,
    network_routes: Vec<NetworkRoute>,
    top_level: HashMap<String, TomlValue>,
    other_tables: HashMap<String, TomlValue>,
}
```

要求：

- 未识别字段必须保留，后续写入不能无意删除。
- `raw_text` 只用于 diff 和回滚，不进入日志。
- `content_hash` 用于防止并发覆盖。

## 3. CodexProfile

```rust
struct CodexProfile {
    name: String,
    model_provider: Option<String>,
    model: Option<String>,
    sandbox: Option<SandboxMode>,
    approval: Option<ApprovalMode>,
    features: Vec<String>,
    env_refs: Vec<SecretRef>,
    mcp_refs: Vec<String>,
    network: Option<String>,
    is_active: bool,
    is_default: bool,
    note: Option<String>,
}
```

校验规则：

- `model_provider` 存在时必须能在 `model_providers` 中找到。
- `mcp_refs` 中每个 name 必须能在 `mcp_servers` 中找到。
- `network` 存在时必须能在 `network_routes` 中找到。
- 切换模型时切换的是完整 profile，不是只改裸 `model` 字段。
- 首次新增第三方 provider 时，需要保留官方订阅 profile。

## 4. ModelProvider

```rust
struct ModelProvider {
    name: String,
    kind: ProviderKind,
    channel: ProviderChannel,
    base_url: Option<String>,
    wire_api: WireApi,
    request_timeout: Option<u32>,
    api_key_ref: Option<SecretRef>,
    http_headers: HashMap<String, String>,
    is_active: bool,
    note: Option<String>,
}

enum ProviderKind {
    OpenAISubscription,
    OpenAIOfficialApi,
    OpenAICompatibleApi,
    LocalGateway,
    CliProxyApi,
    CodexProxy,
}

enum ProviderChannel {
    Subscription,
    Api,
    Gateway,
}

enum WireApi {
    Chat,
    Responses,
    SseStream,
    Custom(String),
}
```

校验规则：

- `channel = Api` 时必须提供 `api_key_ref`。
- `channel = Subscription` 时禁止配置 `api_key_ref`，也禁止读取或复用官方账号 token。
- `channel = Gateway` 时 `base_url` 必须指向本地或显式配置的 gateway。
- secret 只允许通过 env 或 Keychain 引用，不写明文。

## 5. GatewayRuntime

```rust
struct GatewayRuntime {
    id: String,
    source: GatewaySource,
    status: RuntimeStatus,
    host: String,
    port: u16,
    pid: Option<u32>,
    started_at: Option<DateTime<Utc>>,
    local_url: String,
    lan_urls: Vec<String>,
    health_endpoint: String,
    health_ok: bool,
    runtime_dir: PathBuf,
    config_path: PathBuf,
    log_path: PathBuf,
    last_error: Option<String>,
}

enum GatewaySource {
    Embedded,
    ExternalOpenCodexCheckout,
    CodexProxy,
    CliProxyApi,
}

enum RuntimeStatus {
    Stopped,
    Starting,
    Running,
    Degraded,
    Failed,
}
```

要求：

- `Embedded` 是目标形态。
- `ExternalOpenCodexCheckout` 只作为过渡期兼容，不允许硬编码个人路径作为长期方案。
- 默认监听 `127.0.0.1`。
- LAN 模式必须依赖 `GatewayAccess.password_configured = true`。
- 日志记录 route、status、latency、requestId 等摘要，不记录 prompt、文件正文、token。

## 6. GatewayAccess

```rust
struct GatewayAccess {
    local_enabled: bool,
    lan_enabled: bool,
    password_configured: bool,
    auth_scheme: GatewayAuthScheme,
    cookie_policy: CookiePolicy,
    login_rate_limit: RateLimit,
    mobile_url: Option<String>,
    mobile_url_reachable: bool,
}

enum GatewayAuthScheme {
    NoneLocalOnly,
    Sha256Password,
}
```

要求：

- 本机访问可以无密码，但 LAN 访问必须有密码。
- 密码 hash 可以落盘，明文密码不能落盘。
- session token 只保存在内存或安全 cookie，不写配置文件。
- 登录失败必须有 rate limit。

## 7. CodexRuntimeStatus

```rust
struct CodexRuntimeStatus {
    codex_home: PathBuf,
    codex_cli_path: Option<PathBuf>,
    codex_desktop_path: Option<PathBuf>,
    desktop_installed: bool,
    cli_available: bool,
    config_readable: bool,
    auth_state_detected: bool,
    official_resource_scan: OfficialResourceScan,

struct OfficialResourceScan {
    enabled: bool,
    readonly: bool,
    app_asar_path: Option<PathBuf>,
    cache_dir: Option<PathBuf>,
    last_scan_error: Option<String>,
}
```

要求：

- 默认只做安装检测和只读状态识别。
- 不默认解包 `app.asar`。
- 如后续读取官方资源，必须白名单、缓存隔离、路径越界防护。
- 不修改 Codex Desktop 内部文件。

## 8. McpServer

```rust
struct McpServer {
    name: String,
    command: Option<String>,
    args: Vec<String>,
    url: Option<String>,
    env: HashMap<String, String>,
    enabled: bool,
    last_health: Option<HealthStatus>,
    last_error: Option<String>,
}
```

校验规则：

- stdio server 的 `command` 必须在 `$PATH` 中可执行。
- HTTP server 的 `url` 必须可解析。
- env 中疑似 secret 字段必须脱敏。

## 10. ConfigChangePreview

```rust
struct ConfigChangePreview {
    operation: ConfigOperation,
    current_hash: String,
    proposed_text: String,
    diff_lines: Vec<DiffLine>,
    warnings: Vec<String>,
}

enum ConfigOperation {
    SetActiveProfile { name: String },
    CreateProfile(CodexProfile),
    UpdateProfile(CodexProfile),
    DeleteProfile { name: String },
    CreateProvider(ModelProvider),
    UpdateProvider(ModelProvider),
    DeleteProvider { name: String },
    RollbackBackup { backup_id: String },
}
```

写入链路：

1. 读取当前 `config.toml`
2. 校验 `content_hash`
3. 生成 preview 和 diff
4. 用户确认
5. 创建 backup
6. 写 `.tmp`
7. `rename`
8. 失败则 rollback

## 13. SecretRef

```rust
struct SecretRef {
    source: SecretSource,
    ref_id: String,
    display: String,
}

enum SecretSource {
    Env,
    Keychain,
    InlineMemoryOnly,
}
```

红线：

- `InlineMemoryOnly` 只允许存在于当前进程内存。
- 真实 secret 不写日志、不写诊断报告、不写 JSON 导出。
- 写入 config 时只写 `${ENV_VAR}` 或等价引用，不写明文。
