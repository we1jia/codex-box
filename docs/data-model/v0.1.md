## 1. 总览

Codex Box 围绕 8 个核心数据模型构建：

```
CodexConfigSnapshot  ─┬─>  CodexProfile
                       ├─>  ModelProvider
                       ├─>  NetworkRoute
                       ├─>  McpServer
                       └─>  BackupRecord

HealthStatus（聚合状态）      SecretRef（凭据引用）
```

## 2. CodexConfigSnapshot

> 一次 `~/.codex/config.toml` 读取的结构化快照

```rust
struct CodexConfigSnapshot {
    /// 文件路径，默认 ~/.codex/config.toml
    path: PathBuf,

    /// 读取时间
    read_at: DateTime<Utc>,

    /// 原始 TOML 文本（用于 diff 和回滚）
    raw_text: String,

    /// 解析后的结构
    parsed: ParsedConfig,

    /// SHA-256，用于检测变更
    content_hash: String,

    /// 是否为合法 TOML
    valid: bool,

    /// 解析错误（如有）
    parse_errors: Vec<ParseError>,
}

struct ParsedConfig {
    /// 当前 active profile 名
    active_profile: Option<String>,

    /// 所有 profile
    profiles: Vec<CodexProfile>,

    /// 所有 model provider
    model_providers: Vec<ModelProvider>,

    /// 所有 MCP server
    mcp_servers: Vec<McpServer>,

    /// 默认 network route
    default_network: Option<String>,

    /// 所有 network route
    network_routes: Vec<NetworkRoute>,

    /// 顶层自由字段（sandbox / approval / features / env 等）
    top_level: HashMap<String, TomlValue>,
}
```

## 3. CodexProfile

```rust
struct CodexProfile {
    /// profile 名
    name: String,

    /// 引用的 model_provider
    model_provider: String,

    /// 具体模型名，模型选择器展示的是 profile，但调用时仍需要 model
    model: Option<String>,

    /// sandbox 模式: read-only / workspace-write / danger-full-access
    sandbox: SandboxMode,

    /// approval 模式: never / on-failure / on-request / untrusted
    approval: ApprovalMode,

    /// 启用的 features: web / shell / unstaged / ...
    features: Vec<String>,

    /// 引用的环境变量（SecretRef，不存明文）
    env_refs: Vec<SecretRef>,

    /// 引用的 MCP server 名
    mcp_refs: Vec<String>,

    /// 引用的 network route 名
    network: Option<String>,

    /// 是否为默认 profile
    is_default: bool,

    /// 备注
    note: Option<String>,
}

enum SandboxMode {
    ReadOnly,
    WorkspaceWrite,
    DangerFullAccess,
}

enum ApprovalMode {
    Never,
    OnFailure,
    OnRequest,
    Untrusted,
}
```

**校验规则**：
- `model_provider` 必须在 `model_providers` 列表中存在
- `mcp_refs` 中每个 name 必须在 `mcp_servers` 列表中存在
- `network` 必须在 `network_routes` 列表中存在
- 至少一个 profile `is_default = true`

## 4. ModelProvider

```rust
struct ModelProvider {
    /// provider 名
    name: String,

    /// 类型
    kind: ProviderKind,

    /// 通道来源。subscription 只做状态识别，不复用官方账号 token
    channel: ProviderChannel,

    /// base URL
    base_url: String,

    /// API 协议
    wire_api: WireApi,

    /// 请求超时（秒）
    request_timeout: Option<u32>,

    /// API key 引用（SecretRef，不存明文）
    api_key_ref: Option<SecretRef>,

    /// 额外的 HTTP headers
    http_headers: HashMap<String, String>,

    /// 是否为当前 active
    is_active: bool,

    /// 备注
    note: Option<String>,
}

enum ProviderKind {
    OpenAISubscription,
    OpenAIOfficialApi,
    OpenAICompatibleApi,
    LocalGateway,
    CLIPoxyAPI,
    CodexProxy,
}

enum ProviderChannel {
    Subscription,
    Api,
    Gateway,
}

enum WireApi {
    Chat,
    Responses,
    SseStream,
    Custom(String),
}
```

**校验规则**：
- `base_url` 必须可被 `url::Url::parse` 解析
- 当 `channel = Api` 时 `api_key_ref` 必须存在
- 当 `channel = Subscription` 时禁止配置 `api_key_ref`，也禁止读取或复用官方账号 token
- `api_key_ref` 永远不写到日志/快照 raw 文本外

### 4.1 ProtocolAdapter

> M2 只做配置建模；真实请求/响应转换在 M5 Gateway 体验增强中实现。

```rust
struct ProtocolAdapter {
    /// adapter 名，如 openai-chat、openai-responses、local-gateway
    name: String,

    /// 输入协议
    request_format: WireApi,

    /// 输出协议
    response_format: WireApi,

    /// 是否支持流式输出
    supports_stream: bool,

    /// 是否支持 tool calls
    supports_tool_calls: bool,

    /// 是否支持 reasoning 字段
    supports_reasoning: bool,
}
```

**转换目标**：
- Chat Completions / Responses API / 自定义 gateway 请求统一映射
- SSE stream、tool calls、reasoning、usage、error 统一回 Codex Box 内部结构

## 5. NetworkRoute

```rust
struct NetworkRoute {
    /// route 名
    name: String,

    /// 类型
    kind: NetworkKind,

    /// 代理 URL（HTTP/SOCKS 专用）
    proxy_url: Option<String>,

    /// 代理认证
    proxy_auth: Option<ProxyAuth>,

    /// Clash / Mihomo profile 路径
    clash_profile_path: Option<PathBuf>,

    /// 不走代理的地址列表
    no_proxy: Vec<String>,

    /// 是否为当前 active
    is_active: bool,
}

enum NetworkKind {
    /// 直连
    Direct,
    /// HTTP proxy
    HttpProxy,
    /// SOCKS5 proxy
    Socks5Proxy,
    /// Clash / Mihomo 配置文件
    ClashProfile,
    /// 自定义
    Custom(String),
}

struct ProxyAuth {
    username: String,
    password_ref: SecretRef,
}
```

**校验规则**：
- `kind = HttpProxy / Socks5Proxy` 时 `proxy_url` 必须存在
- `kind = ClashProfile` 时 `clash_profile_path` 必须存在且可读
- 任何 route 都不能设置系统全局代理（红线）

## 6. McpServer

```rust
struct McpServer {
    /// server 名
    name: String,

    /// 启动命令
    command: String,

    /// 启动参数
    args: Vec<String>,

    /// 环境变量
    env: HashMap<String, String>,

    /// 是否启用
    enabled: bool,

    /// 最近一次健康状态
    last_health: Option<HealthStatus>,

    /// 最近一次错误信息（已脱敏）
    last_error: Option<String>,
}
```

**校验规则**：
- `command` 必须在 `$PATH` 中可执行
- `env` 中的 secret 字段必须用 `SecretRef`（红线）
- 启动失败时 `last_error` 必须脱敏

## 7. BackupRecord

```rust
struct BackupRecord {
    /// 备份 ID（UUID）
    id: String,

    /// 备份时间
    created_at: DateTime<Utc>,

    /// 备份文件路径
    file_path: PathBuf,

    /// 触发原因: manual / pre-write / scheduled / pre-rollback
    reason: BackupReason,

    /// 原始 config 的 content_hash
    content_hash: String,

    /// 字节大小
    size_bytes: u64,

    /// 备注
    note: Option<String>,
}

enum BackupReason {
    Manual,
    PreWrite,
    PreRollback,
    Scheduled,
}
```

**存储路径**：`~/.codex/codex-box/backups/{timestamp}-{hash}.toml`

**保留策略**：
- 最多保留 50 个
- 超过 7 天的允许清理
- 清理操作本身也要写 audit log

## 9. SecretRef

> 凭据引用，**永远不存明文**

```rust
struct SecretRef {
    /// 来源: env 变量 / keychain / 自填
    source: SecretSource,

    /// 来源标识
    /// - Env("OPENAI_API_KEY")  → 引用环境变量
    /// - KeychainEntry("openai-official")  → 引用 keychain
    /// - Inline("sk-...")  → ⚠️ 仅在本地缓存中使用，不允许落盘
    ref_id: String,

    /// 显示用的脱敏串
    display: String,  // 例: "sk-proj-***"
}

enum SecretSource {
    Env,
    Keychain,
    Inline,  // ⚠️ 临时内存
}
```

**红线**：
- ❌ `SecretRef` 的真实值**永远不写**到日志、UI、JSON 导出、备份文件
- ✅ UI 一律显示 `display` 字段
- ✅ 写入 config.toml 时只写入 `${ENV_VAR}` 形式的引用，不写入明文
- ✅ backup 文件保存原始的 `${ENV_VAR}` 引用形式

## 11. 序列化

### TOML 写出顺序
1. `[model_providers.xxx]` 区段（先于 profile 引用）
2. `[mcp_servers.xxx]` 区段
3. `[network.xxx]` 区段
4. `[profile.xxx]` 区段
5. 顶层字段（sandbox / approval / features / env / 注释）

### TOML 风格
- 缩进：2 空格
- key 顺序：固定（见上）
- 注释：用 `#` 单行，保留用户原始注释
- 不引入 tab、不引入多余空行
