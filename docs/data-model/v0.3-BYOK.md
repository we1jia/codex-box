## 1. 总览

Codex Box v0.3 围绕三组模型构建:

1. **Codex 配置模型**:`CodexConfigSnapshot`、`CodexProfile`、`ModelProvider`、`McpServer`、`NetworkRoute`
2. **BYOK 模型目录模型**:`ModelCatalogEntry`、`ProviderRoute`、`OpenCodexCustomConfig`
3. **运行时 / 安全写入 / 诊断模型**:`CodexRuntimeStatus`、`ConfigChangePreview`、`BackupRecord`、`HealthStatus`、`SecretRef`

```
CodexConfigSnapshot
  ├─ CodexProfile
  ├─ ModelProvider
  ├─ McpServer
  └─ NetworkRoute

OpenCodexCustomConfig
  ├─ ProviderRoute[]
  └─ ModelCatalogEntry[]

ConfigChangePreview
  ├─ BackupRecord
  ├─ HealthStatus
  └─ SecretRef
```

## 2. CodexConfigSnapshot

```rust
struct CodexConfigSnapshot {
    path: PathBuf,                    // ~/.codex/config.toml
    read_at: DateTime<Utc>,
    raw_text: String,
    parsed: ParsedConfig,
    content_hash: String,
    valid: bool,
    parse_errors: Vec<ParseError>,
}

struct ParsedConfig {
    active_profile: Option<String>,
    profiles: Vec<CodexProfile>,
    model_providers: Vec<ModelProvider>,
    mcp_servers: Vec<McpServer>,
    default_network: Option<String>,
    network_routes: Vec<NetworkRoute>,
    top_level: HashMap<String, TomlValue>,
    other_tables: HashMap<String, TomlValue>,
}
```

要求:

- 未识别字段必须保留,后续写入不能无意删除。
- `raw_text` 只用于 diff 和回滚,不进入日志。
- `content_hash` 用于防止并发覆盖。

## 3. CodexProfile

```rust
struct CodexProfile {
    name: String,
    model_provider: Option<String>,
    model: Option<String>,
    sandbox: Option<SandboxMode>,
    approval: Option<ApprovalMode>,
    features: Vec<String>,
    env_refs: Vec<SecretRef>,
    mcp_refs: Vec<String>,
    network: Option<String>,
    is_active: bool,
    is_default: bool,
    note: Option<String>,
}
```

校验规则:

- `model_provider` 存在时必须能在 `model_providers` 中找到。
- `mcp_refs` 中每个 name 必须能在 `mcp_servers` 中找到。
- `network` 存在时必须能在 `network_routes` 中找到。
- 切换模型时切换的是完整 profile,不是只改裸 `model` 字段。
- 首次新增第三方 provider 时,需要保留官方订阅 profile。

## 4. ModelProvider(`~/.codex/config.toml` 内)

```rust
struct ModelProvider {
    name: String,
    kind: ProviderKind,
    channel: ProviderChannel,
    base_url: Option<String>,
    wire_api: WireApi,
    request_timeout: Option<u32>,
    api_key_ref: Option<SecretRef>,
    http_headers: HashMap<String, String>,
    models: Vec<String>,                // 该 provider 下的模型 id 列表
    is_active: bool,
    note: Option<String>,
}

enum ProviderKind {
    OpenAISubscription,
    OpenAIOfficialApi,
    OpenAICompatibleApi,
    LocalGateway,
}

enum ProviderChannel {
    Subscription,                       // 官方订阅,不读 token
    Api,                                // 走 env 引用 key
    Gateway,                            // 走本地/中间层 endpoint
}

enum WireApi {
    Chat,
    Responses,
    SseStream,
    Custom(String),
}
```

校验规则:

- `channel = Api` 时必须提供 `api_key_ref`。
- `channel = Subscription` 时禁止配置 `api_key_ref`,也禁止读取或复用官方账号 token。
- `channel = Gateway` 时 `base_url` 必须指向本地或显式配置的 endpoint。
- secret 只允许通过 env 或 Keychain 引用,不写明文。

## 5. ProviderRoute(`~/.opencodex/providers.json` 条目,AITabby 约定)

```rust
struct ProviderRoute {
    name: String,                       // 唯一 name,与 ModelProvider.name 对应
    base_url: String,
    wire_api: WireApi,
    api_key: Option<String>,            // 文件字段;Codex Box 只允许 "$ENV_VAR" 或 "${ENV_VAR}"
    http_headers: HashMap<String, String>,
    enabled: bool,                      // 是否在 Codex App 模型下拉里可见
    note: Option<String>,
    // 未知字段保留,写入时原样回写
}
```

要求:

- 文件不存在时按"空配置"处理,不要报错。
- 未知字段保留并回写,不删字段。
- AITabby 文件字段名是 `api_key`,Codex Box UI 内部可映射为 `api_key_ref`。
- 写回文件时只允许 `$ENV_VAR` 或 `${ENV_VAR}` 引用,不接受 inline value。
- 与 AITabby 共存时优先写 `{ "providers": [...] }` envelope,不是顶层裸数组。
- Codex Box 写这份 JSON 时走与 `config.toml` 相同的 backup → diff → confirm → atomic write → rollback 闭环。

## 6. ModelCatalogEntry(`~/.opencodex/custom_model_catalog.json` 条目,AITabby 约定)

```rust
struct ModelCatalogEntry {
    slug: String,                       // Codex App picker 看到的 model id
    model: String,                      // 通常等于 slug
    display_name: Option<String>,       // 友好显示名
    provider: String,                   // 归属 provider name
    backend_model: Option<String>,      // 转发给上游 API 的真实 model
    backend_provider: Option<String>,   // 路由到哪个 providers.json provider
    visibility: String,                 // "list" 才进入 /v1/models
    reasoning: Option<ReasoningConfig>,
    note: Option<String>,
    // 未知字段保留
}

struct ReasoningConfig {
    enabled: bool,
    levels: Vec<String>,                // e.g. ["low", "medium", "high"]
}
```

要求:

- AITabby 文件字段名是 `slug` / `backend_model` / `backend_provider` / `visibility`。
- Codex Box UI 内部可把 `slug` 映射成 `model_id`,把 `visibility == "list"` 映射成 `visible=true`,但写回文件必须恢复 AITabby 字段。
- `/v1/models` 返回给 Codex App 的 `id` 默认使用 `slug` 本身。
- `/v1/responses` / `/v1/chat/completions` 转发前必须把请求体 `model` 改写为 `backend_model` 或 `model`,不能把 picker 展示用 `slug` 原样发给上游。
- 同 `ProviderRoute` — 未知字段保留、空文件按空处理、secret 走 env 引用。

## 11. ConfigChangePreview

```rust
struct ConfigChangePreview {
    target: ConfigTarget,               // ~/.codex/config.toml 或 ~/.opencodex/*.json
    operation: ConfigOperation,
    current_hash: String,
    proposed_text: String,
    diff_lines: Vec<DiffLine>,
    warnings: Vec<String>,
}

enum ConfigTarget {
    CodexConfig,                        // ~/.codex/config.toml
    OpenCodexProviders,                 // ~/.opencodex/providers.json
    OpenCodexCatalog,                   // ~/.opencodex/custom_model_catalog.json
}

enum ConfigOperation {
    SetActiveProfile { name: String },
    CreateProfile(CodexProfile),
    UpdateProfile(CodexProfile),
    DeleteProfile { name: String },
    CreateProvider(ModelProvider),
    UpdateProvider(ModelProvider),
    DeleteProvider { name: String },
    ToggleModelVisibility { slug: String, visibility: String },
    UpsertProviderRoute(ProviderRoute),
    DeleteProviderRoute { name: String },
    UpsertCatalogEntry(ModelCatalogEntry),
    DeleteCatalogEntry { slug: String },
    RollbackBackup { backup_id: String },
}
```

写入链路:

1. 读取当前文件
2. 校验 `content_hash`
3. 生成 preview 和 diff
4. 用户确认
5. 创建 backup
6. 写 `.tmp`
7. `rename`
8. 失败则 rollback

## 14. SecretRef

```rust
struct SecretRef {
    source: SecretSource,
    ref_id: String,                      // e.g. "OPENAI_API_KEY"
    display: String,                     // UI 脱敏显示,如 "OPENAI_API_KEY (configured)"
}

enum SecretSource {
    Env,                                // ${ENV_VAR}
    Keychain,
    InlineMemoryOnly,                   // 仅进程内存,不允许写入文件
}
```

红线:

- `InlineMemoryOnly` 只允许存在于当前进程内存。
- 真实 secret 不写日志、不写诊断报告、不写 JSON 导出。
- 写入 config 时只写 `${ENV_VAR}` 或等价引用,不写明文。
- 写入 `~/.opencodex/*.json` 时同上。

## 15. 持久化路径

| 内容 | 路径 | 谁写 |
|---|---|---|
| 主 config | `~/.codex/config.toml` | Codex App 自己(Codex Box 写它) |
| Codex Box 自有配置 | `~/.codex/codex-box/config.json` | Codex Box |
| 备份 | `~/.codex/codex-box/backups/{ts}-{hash}.{toml,json}` | Codex Box |
| 日志 | `~/.codex/codex-box/logs/{date}.log` | Codex Box |
| 诊断报告 | `~/.codex/codex-box/reports/{date}.md` | Codex Box |
| Provider 路由 | `~/.opencodex/providers.json` | Codex Box + AITabby CLI 共享 |
| 自定义模型目录 | `~/.opencodex/custom_model_catalog.json` | Codex Box + AITabby CLI 共享 |

## 16. 与 v0.2 数据模型的关键差异

| v0.2 | v0.3 | 原因 |
|---|---|---|
| `GatewayRuntime` / `GatewayAccess` | 删除 | 项目不再起外部 gateway 进程 |
| `GatewaySource::ExternalOpenCodexCheckout` | 删除 | 不再 spawn 外部 OpenCodex |
| `~/.codex/codex-box/opencodex/runtime/` | 删除 | 没有"外部 OpenCodex runtime"概念 |
| 无 | `ProviderRoute`(`~/.opencodex/providers.json` 条目) | BYOK 多 provider 路由核心模型 |
| 无 | `ModelCatalogEntry`(`~/.opencodex/custom_model_catalog.json` 条目) | BYOK 自定义模型目录 |
| 无 | `OpenCodexCustomConfig`(`~/.opencodex/` 完整快照) | 让前端能一次拿全两份 JSON |
| 无 | `ConfigTarget` 枚举 | 写入闭环对多个文件都生效 |
| 无 | `HealthCheck::OpenCodexProvidersValid` / `OpenCodexCatalogValid` | 诊断新加的 JSON 完整性检查 |
| 无 | `HealthCheck::ProviderUrlReachable` / `ProviderEnvExists` | 诊断 provider 可用性 |
